# Vulfram Glossary (vNext)

## Host

External application that drives Vulfram commands and frame lifecycle.

## Core

Rust engine side (`vulfram-runtime` + dependent crates).

## Realm

Logical scene/render domain (`realm-3d` or `realm-2d`).

A realm does not own fixed output resolution.

## Target

Composition destination (`window` or `texture`) that owns clear and layer stack.

## Layer (`TargetLayer`)

Composition instruction that renders one realm into one target region with blend/opacity/order.

## Texture Target

A target of kind `texture` that produces sampleable output for other realms.

## RenderInvocation

One concrete execution of one realm for one target layer with resolved pixel rect and render size.

## FrameGraph

Global frame scheduler that orders targets by produced/consumed texture dependencies.

## Graph3D / Graph2D

Per-invocation internal pass graphs for 3D and 2D realms.

## Pass

Customizable graph unit with `input`, `output`, `require`, `params`, `priority`, and shader logic.

## Require

Hard dependency list for pass execution. Missing required resources skip the pass.

## Shader DSL

Restricted client-facing shader subset; physical WGSL binding details are generated by the core.

## ShaderMaterial

Unified material model with closed presets `standard` and `pbr` in current phase.

## Deterministic Ordering

Stable ordering across identical frames by dependency rules and explicit priorities.
